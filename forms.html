Form Tags Name Purpose
<form> Form The form element collects information and sends it somewhere for processing.
<form action="/example.html"> action The action attribute determines where the information is sent.
<form method="POST"> method The method attribute is assigned a HTTP verb that is included in the HTTP request.
<input> Input It creates an input field in our form.
<input type="text"> type The type attribute determines how the input field is rendered on the webpage and what kind of data it can accept. Its default value is "text".
<input name="firstname"> name The name attribute acts as a label for the data we enter in the input field.
<input value="bryan"> value The value attribute contains what the user types into the field, and then it is linked with the name attribute to be sent as a key:value pair.
<label> Label The label element has opening and closing tags and it is used to properly identify an input field.
<label for="meal"> for To associate a label with an input field we use the for attribute and set it equal to the value of the input's id attribute.
<input id="meal"> id To associate a label with an input field we use the for attribute and set it equal to the value of the input's id attribute.
<input type="password"> password The type="password" creates a password field that obscures the information we type into it, usually replacing text with asterisks.
<input type="number"> number The type="number" creates an input field that only accepts number and a few special characters like: +, - and .
<input type="number" step="1"> step The step attribute in a type="number" field allows us to add arrows to increase/decrease the number by the value of the step attribute.
<input type="range" min="0" max="100"> range The type="range" creates a slider to which we can add a minimum and maximum value using the min and max attributes.
<input type="range" step="5"> range In the type="range" we can also add the step attribute which will determine how smooth and fluid the slider will move.
<input type="checkbox" name="topping"> checkbox The type="checkbox" allows us to present several options to the user, in which case we need to add the same value for the name attribute to all the checkboxes. Each <input type="checkbox"> needs to have an associated label.
<input type="radio" name="answer"> radio We use radio buttons to present several options but only allow one possible answer. To group several radio buttons we need to assign the same value to the attribute name.
<select> Select The select element creates a dropdown list that we can then populate by adding multiple option elements.
<select id="lunch" name="lunch"> Select In order to create the dropdown list, the <select> element needs to have an id attribute to link it with a <label> element, and a name attribute to link it with multiple <option> elements.
<option value="pizza"> Pizza </option> Option The <option> element is used to add options to the dropdown list, it needs to have a value attribute that needs to be equal to the text between the opening and closing <option> tags
<textarea id="blog" name="blog" rows="5" cols="30"> </textarea> Textarea The <textarea> element is used to create a text input field larger than the <input type="text"> that we previously studied, and we can determine the size of the text field by using the rows and cols attributes.
<input type="submit" value="Send"> submit The type="submit" creates a submit button in our form. We can change the default "submit" word that is shown in the button by adding a different word in the value attribute.
